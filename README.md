## Практическое задание 7 — Базовый сервер с аутентификацией

Серверное приложение на Node.js + Express, реализующее регистрацию и вход пользователей с хешированием паролей (bcrypt). Данные хранятся в памяти.

**Технологии:** Node.js, Express, bcrypt, nanoid, cors

### API эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| `POST` | `/api/auth/register` | Регистрация нового пользователя |
| `POST` | `/api/auth/login` | Вход (проверка пароля) |

---

## Практическое задание 8 — JWT-авторизация

Расширение сервера: добавлена выдача JWT-токена при успешном входе, защищённые маршруты для управления товарами (`/api/products`). Токен проверяется middleware `authMiddleware`.

**Технологии:** Node.js, Express, jsonwebtoken, bcrypt, nanoid, cors

### Новые эндпоинты

| Метод | URL | Защита | Описание |
|-------|-----|--------|----------|
| `GET` | `/api/auth/me` | Требуется токен | Получение данных текущего пользователя |
| `GET` | `/api/products` | Нет | Список товаров (доступен всем) |
| `POST` | `/api/products` | Требуется токен | Создание товара |
| `PUT` | `/api/products/:id` | Требуется токен | Полное обновление товара |
| `DELETE` | `/api/products/:id` | Требуется токен | Удаление товара |

## Практическое задание 9 — Refresh-токены

Внедрена пара токенов: **access-токен** (срок жизни 15 минут) и **refresh-токен** (срок жизни 7 дней). Добавлен эндпоинт `/api/auth/refresh` для ротации токенов. Refresh-токены хранятся на сервере в `Set`, при ротации старый удаляется, выдаётся новый.

**Технологии:** Node.js, Express, jsonwebtoken, bcrypt, nanoid, cors

### Изменения в API

| Метод | URL | Описание |
|-------|-----|----------|
| `POST` | `/api/auth/login` | Возвращает `accessToken` и `refreshToken` |
| `POST` | `/api/auth/refresh` | Принимает `refreshToken`, возвращает новую пару токенов |
| `POST` | `/api/auth/logout` | Удаляет refresh-токен из хранилища (опционально) |

## Практическое задание 10 — React-клиент с Axios Interceptors

Разработано полноценное SPA на React, подключённое к серверу. Реализованы страницы входа, регистрации и управления товарами. Использованы **Axios interceptors** для автоматического добавления токена в заголовки и для обновления access-токена при получении ошибки 401.

**Технологии:** React 19, Vite, Axios, react-router-dom, CSS modules

### Особенности клиента

- **Страница товаров:** публичная (гости видят список, но не могут добавлять/редактировать).
- **Авторизация:** сохранение `accessToken` и `refreshToken` в `localStorage`, автоматическая подстановка токена в запросы.
- **Interceptors:** при ответе 401 отправляется запрос на `/api/auth/refresh`, после чего повторяется исходный запрос.
- **Защита маршрутов:** страница `/users` доступна только администратору.

## Практическое задание 11 — Роли и права доступа

Добавлена система ролей (RBAC). Реализованы роли: **Гость**, **Пользователь**, **Продавец**, **Администратор**. Права доступа разграничены на сервере (middleware `roleMiddleware`) и на клиенте (условный рендеринг).

### Роли и их возможности

| Роль | Права |
|------|-------|
| **Гость** | Просмотр товаров |
| **Пользователь** | Просмотр товаров, просмотр своего профиля |
| **Продавец** | Просмотр, создание, редактирование товаров |
| **Администратор** | Все права продавца + управление пользователями (список, редактирование, блокировка) |

### API эндпоинты (обновлённые)

| Маршрут | Метод | Доступ | Описание |
|---------|-------|--------|----------|
| `/api/auth/register` | `POST` | Гость | Регистрация |
| `/api/auth/login` | `POST` | Гость | Вход |
| `/api/auth/refresh` | `POST` | Гость | Обновление токенов |
| `/api/auth/me` | `GET` | Пользователь | Профиль |
| `/api/users` | `GET` | Администратор | Список пользователей |
| `/api/users/:id` | `GET` | Администратор | Получить пользователя по ID |
| `/api/users/:id` | `PUT` | Администратор | Обновить пользователя (роль, блокировка) |
| `/api/users/:id` | `DELETE` | Администратор | Удалить пользователя |
| `/api/products` | `GET` | Гость, Пользователь | Список товаров |
| `/api/products/:id` | `GET` | Гость, Пользователь | Товар по ID |
| `/api/products` | `POST` | Продавец, Администратор | Создать товар |
| `/api/products/:id` | `PUT` | Продавец, Администратор | Обновить товар |
| `/api/products/:id` | `DELETE` | Администратор | Удалить товар |

### Клиентская часть

- На странице товаров кнопки «Добавить», «Редактировать», «Удалить» отображаются только для авторизованных пользователей с соответствующей ролью.
- Администратор имеет доступ к странице `/users`, где может просматривать список пользователей, менять их роли и блокировать.
- В шапке приложения отображается роль текущего пользователя и кнопка выхода.
